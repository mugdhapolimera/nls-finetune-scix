SSE stands for Streaming SIMD Extensions. SIMD stands for single instruction, multiple data. These are the instruction set that allow for vectorization to occur on the CPU. Normally, the compiler will translate code into these instructions when working on arrays. The SSE instruction set was developed by intel, and SSEx.x are different versions of SSE (SSE, SSE2, SSE3, SSE4).

The \`-mia32\` argument tells the compiler to only generate code for x86/x87, without any vector instructions. It might be that that processor doesn’t support later versions of SSE. I believe I can look up what instruction sets intel processors use. I’ll get back to you on if that processor does or not. Also, the compute nodes may have a different processor.

You will get a performance hit with \`-mia32\`, since it wont vectorize. You might want to try compiling with other SSE versions and see which ones work. I’d start with \-mavx, and move down the list.
